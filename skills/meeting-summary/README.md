# meeting-summary

Processes a meeting transcript and produces a structured `summary.md` — decisions, action items, key insights, and next steps in a consistent format.

Includes a bundled script (`scripts/fetch_transcript.py`) that pulls the latest meeting transcript directly from **Fathom** and saves it in the right folder automatically.

Works with any transcript source: use the Fathom script, paste text directly, or point to an existing file.

## How to use

```
/meeting-summary
```

Then either:
- Say "meeting just ended" and the skill runs the Fathom script automatically
- Paste the transcript text directly and tell it the meeting name/date
- Point it at an existing transcript file

## Setup (Fathom)

The bundled `scripts/fetch_transcript.py` requires two environment variables in your `.env` file:

```
FATHOM_API_KEY=your_fathom_api_key_here
FATHOM_RECORDED_BY=you@yourcompany.com
```

**`FATHOM_API_KEY`** — Get this from your Fathom account:
1. Go to [fathom.video](https://fathom.video) → Settings → API
2. Generate a new API key
3. Paste it into your `.env`

**`FATHOM_RECORDED_BY`** — The email address on your Fathom account. The script uses this to filter meetings recorded by you (not all meetings in your workspace).

> Make sure `.env` is in your `.gitignore` — never commit your API key.

## Install dependencies

```bash
pip install requests python-dotenv
```

## Folder structure (suggested)

The script saves transcripts following this convention:

```
meetings/
└── YYYY/
    └── MM/
        └── DD/
            └── [meeting-title]/
                ├── transcript.md    ← saved by the script
                └── summary.md       ← generated by this skill
```

## Not using Fathom?

If you use a different service (Otter, Fireflies, etc.) or paste transcripts manually, the script is optional — the skill works fine without it. Just save your transcript as `transcript.md` in the correct folder and invoke the skill.
