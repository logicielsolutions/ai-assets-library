# Meeting Summary — Extract & Summarize Meeting Transcripts

This skill processes meeting transcripts and produces a structured `summary.md` file saved alongside the transcript.

## Suggested Folder Convention

```
meetings/
└── YYYY/
    └── MM/
        └── DD/
            └── [meeting-title]/
                ├── transcript.md    ← input
                └── summary.md       ← output (generated by this skill)
```

Adapt this structure to whatever convention your project uses.

## Workflow

### Step 1: Locate the Transcript

**If you use Fathom:**
- Run `python skills/meeting-summary/scripts/fetch_transcript.py` from your project root
- This fetches the most recent Fathom meeting and saves `transcript.md` in the correct folder
- Note the saved path from the script output, then read that `transcript.md` and proceed to Step 2
- Requires `FATHOM_API_KEY` and `FATHOM_RECORDED_BY` set in your `.env` — see the skill README for setup

**If you use another transcript service (Otter, Fireflies, etc.):**
- Fetch or export the transcript and save it as `transcript.md` in the correct folder, then proceed to Step 2

**Otherwise:**
- The user will specify a date and/or meeting name. Find the matching folder.
- If the user pastes a transcript directly, ask which date and meeting name it belongs to, then save it as `transcript.md` in the correct folder (create the folder if needed).
- If `transcript.md` already exists in the specified folder, read it.
- If the folder or transcript doesn't exist and the user hasn't pasted one, ask them to provide it.

### Step 2: Analyze the Transcript

Read the full transcript carefully. Identify:

1. **Participants** — who spoke, their roles/affiliations
2. **Recording link** — usually at the top of transcripts
3. **Duration** — from the header or last timestamp
4. **Topics discussed** — group the conversation into logical themes
5. **Decisions** — concrete agreements ("let's do X", "we'll go with Y")
6. **Action items** — look for lines marked `ACTION ITEM:` AND implicit commitments in conversation ("I will...", "let me...", "you should..."). For each, identify the owner and what they need to do.
7. **Insights** — non-obvious signals: sentiments, concerns raised, things said between the lines
8. **Open questions** — things left unresolved or explicitly deferred
9. **Next meeting** — when they plan to meet again and what's expected by then

### Step 3: Generate summary.md

Write the summary using the following template:

```markdown
# Meeting Summary: [Participant A] <> [Participant B]

**Date:** DD-MM-YYYY
**Duration:** ~X mins
**Participants:** Name (Role/Company), Name (Role/Company)
**Recording:** [Link](url)

---

## Context

> 1-2 sentences on why this meeting happened and what stage of work it relates to.

## Key Topics Discussed

- **Topic name** — brief description of what was discussed
- **Topic name** — brief description of what was discussed
- ...

## Decisions Made

- Decision description — with enough context to understand it standalone
- ...

## Action Items

| # | Owner | Action | Deadline/Priority |
|---|-------|--------|-------------------|
| 1 | Name  | What they need to do | By when / ASAP / TBD |
| 2 | Name  | What they need to do | By when / ASAP / TBD |

## Key Insights

- Insight or observation worth remembering
- ...

## Open Questions

- Question or unresolved topic
- ...

## Next Meeting

- **When:** Date or timeframe
- **Expected by then:** What each participant should have done or prepared
```

### Step 4: Save

Write the file directly as `summary.md` in the same folder as the transcript. Do NOT ask for preview or approval — just create the file. The user will review it themselves.

## Important Notes

- Transcripts may contain transcription errors, garbled text, or mixed languages. Use context to interpret meaning — don't reproduce garbled text in the summary.
- Look for implicit commitments in the full conversation, not just auto-detected action item lines.
- Keep the summary concise but complete — it should be useful months later without re-reading the transcript.
- Use professional, clear language even if the conversation was casual.
- Do NOT editorialize or add opinions — stick to what was actually said.
